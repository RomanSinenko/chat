# Проект
Backend MVP чат-приложения с поддержкой realtime обмена сообщениями по WebSocket.  
Проект разрабатывается с нуля с прицелом на масштабирование, авторизацию по SMS и последующее добавление end-to-end шифрования (E2EE).

---

# Цель проекта
Создать минимально работающий backend чата (MVP), который умеет:
- поднимать HTTP API
- принимать WebSocket-подключения
- поддерживать realtime обмен сообщениями между пользователями
- обрабатывать ошибки формата сообщений и недоступного получателя
- хранить сообщения (следующий этап)

С дальнейшим развитием:
- авторизация и регистрация по SMS-коду
- хранение пользователей, устройств и сообщений
- масштабирование через Redis / брокеры сообщений
- добавление E2EE (например, через Signal Protocol или libsodium)

# Архитектурная идея
Проект строится поэтапно, от простого к расширяемому.

Слои:
- HTTP API — для обычных запросов
- WebSocket — для realtime соединений
- ConnectionManager — для управления активными WebSocket-подключениями
- DB layer — для хранения данных
- Auth layer — для SMS-авторизации
- Encryption layer — для будущего E2EE

Ключевые принципы:
- сначала минимально рабочий MVP
- затем постепенное усложнение
- не писать заранее лишнюю абстракцию
- каждую новую часть понимать, а не просто копировать
- выносить ответственность из endpoint в отдельные методы и классы

# Текущее поведение системы

1. Пользователь подключается:
   `/ws/1`

2. Получает:
   `Подключение с сервером установлено!`

3. Отправляет:
   `2:hello`

4. Пользователь 2 получает:
   `From: 1: hello`

5. Если пользователь не найден:
   `User 2 not connected`

6. Если формат неверный:
   сервер отправляет сообщение об ошибке формата

7. Если вместо номера пользователя передан нечисловой текст:
   сообщение игнорируется без падения сервера

# Текущее состояние архитектуры

Сейчас WebSocket-логика разделена на 2 части:

- `websocket_endpoint`:
  - принимает подключение
  - получает входящие сообщения
  - проверяет формат
  - вызывает методы manager

- `ConnectionManager`:
  - хранит активные соединения
  - подключает пользователя
  - отключает пользователя
  - отправляет сообщения другому пользователю
  - отправляет сообщения текущему клиенту

Это первый шаг к более чистой архитектуре без глобального словаря подключений.
